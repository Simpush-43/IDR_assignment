# Institute of Digital Risk (IDR) - Brand Identity & Homepage

Welcome to the brand identity and frontend repository for the **Institute of Digital Risk (IDR)**. IDR is an industry-led training and deployment institute purpose-built to bridge the gap between academic knowledge and the practical skills demanded by regulated, high-consequence environments in digital, cyber, and AI risk.

## 🎯 Project Overview

This project includes two main deliverables:
1. **Brand Identity:** A custom logo design provided in two variants (Icon-only and Icon + Text).
2. **Responsive Homepage:** A fully responsive, single-page website built strictly with Vanilla HTML, CSS, and JavaScript.

---

## 🎨 Design Note: Brand & Logo Choices

**Symbolism, Colour, and Typography**
The IDR logo utilizes a geometric, cube-inspired icon to symbolize structure, robust resilience, and the multi-faceted nature of digital risk management. The primary color palette relies on a vibrant, energetic orange paired with stark black and white, establishing a modern, high-tech, and authoritative identity suitable for a premier educational institute. For typography, a clean, minimalist sans-serif font family was chosen to ensure strict legibility across all screen sizes—from small mobile favicons to large desktop headers—reinforcing a professional and forward-thinking brand.

---

## 💻 Technical Features

The homepage was built from scratch following modern web standards, with **no external CSS frameworks** (like Bootstrap or Tailwind).

* **Semantic HTML5:** Proper use of `<header>`, `<nav>`, `<main>`, `<section>`, and `<footer>` tags for accessibility and SEO.
* **Vanilla CSS Layouts:** Extensively uses CSS Flexbox and CSS Grid to create a fluid, mobile-first responsive design that adapts flawlessly to any screen size.
* **Sticky Navigation:** A custom, fixed navigation bar featuring a mobile hamburger menu and smooth scrolling behavior to page sections.
* **Interactive Elements:** Clean hover states for buttons, navigation links, and service cards to enhance user experience.
* **Web Typography:** Integration of Google Fonts for a polished, readable tech/education feel.

---

## 📂 Page Structure

The single-page website is divided into the following key sections:

1. **Hero Section:** Features a strong mission headline ("Advancing the Future of Digital Risk"), a descriptive subheadline, and clear Call-to-Action (CTA) buttons.
2. **About IDR:** Explains IDR's mission as an industry-led institute, emphasizing the bridge between UK university partnerships and real-world project deployment.
3. **Our Model (Services):** A grid-based presentation of IDR's core pillars:
   * *Academy* (Bootcamps & Upskilling)
   * *Innovation & Incubation* (AI Governance & IP Development)
   * *Advisory Services* (NIST, ISO 27001, NIS2)
   * Includes a visual "Train → Hire → Innovate → Deploy" pipeline.
4. **Community:** Highlights the target audience, including university students, early-career pros, and practitioners in critical infrastructure and financial services.
5. **Contact:** A "Register Interest" form allowing users to connect based on their profile type.

## 🚀 How to Run the Project

Since this project uses pure Vanilla HTML, CSS, and JS, no complex build tools or package managers (like npm) are required.

1. Clone or download this repository to your local machine.
2. Ensure all files (`index.html`, `style.css`, `script.js`, and logo images) are in the same root folder.
3. Simply double-click the `index.html` file to open it in your default web browser.
